# User Experience & Installation Guide

## Design Philosophy

### KISS Principles (Keep It Simple, Stupid)
- One obvious way to do each task
- No hidden features or buried menus
- Progressive disclosure â€” show basics first, details on demand
- Sensible defaults that work for 90% of cases

### Avoiding Norman Doors
A "Norman Door" is a door you push when you should pull (or vice versa) â€” bad design that requires instructions. Our app should:

- **Buttons do what they say** â€” "Scan Card" scans a card
- **Visual affordances** â€” Draggable things look draggable, clickable things look clickable
- **No dead ends** â€” Always a clear next step
- **Undo available** â€” Mistakes are recoverable
- **Feedback on every action** â€” User knows something happened

### UX Golden Rules for This App

1. **One primary action per screen** â€” Scan page scans, Price page prices
2. **Big, obvious buttons** â€” Not tiny links
3. **Show, don't tell** â€” Preview images, show data in tables
4. **Confirm destructive actions** â€” "Delete card? This cannot be undone."
5. **Remember user's work** â€” Auto-save, don't lose data on crash
6. **Error messages that help** â€” "API key invalid" not "Error 401"
7. **Progress indicators** â€” Spinners, progress bars, "Step 2 of 4"

- Side panels become full-screen overlays
- Touch-friendly button sizes (min 44x44px)
